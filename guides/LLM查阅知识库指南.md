# LLM 查阅知识库指南

> 本指南供 LLM 从 MBTI 知识库中检索和提取信息使用。
> LLM 的角色是**内容提取者**：定位相关页面、提取信息、客观呈现。
> 查阅过程中不修改任何已有页面，仅在必要时做最轻量的记录。

---

## 一、检索路径

Wiki 页面采用严格层级链接结构，每个页面只链接到直接父级和同类别兄弟页面，不存在跨类别链接。检索时沿层级下钻即可精准定位。

```
第一级：wiki/wiki-index.md（语义目录）
  → 通过摘要关键词匹配目标分类
  → 类型相关：定位到枢纽页（如 [[ENFJ]]）
  → 共享内容：直接定位 dimensions/ / type-groups/ / strategies/ / theory/

第二级：枢纽页（类型内导航）
  → 枢纽页列出该类型所有详情页和关系页的链接
  → 通过枢纽页定位具体详情页

第三级：详情页（具体内容）
  → 每个详情页开头有 `> ` 引用格式的一句话摘要
  → 读取摘要即可判断是否需要阅读全文
  → 相关页面区块仅含父级和同系列页面，不会引导到无关分支

第四级：知识单元（Wiki 不足时）
  → units/index.md → 定位来源文件夹 → 来源 index.md → 通过 type + category + tags 定位
  → 最多阅读 5 个相关知识单元

不可访问：raw/（原始资料层）
```

### 摘要快速判断

每个 Wiki 页面在 frontmatter 之后、正文标题之前都有一行 `> ` 格式的摘要。跳转到一个页面后，**先读摘要再决定是否阅读全文**，可大幅减少无效阅读。

## 二、查阅深度控制

### 按需求性质选择深度

| 模式 | 触发条件 | 查阅规则 |
|------|----------|----------|
| **精确提取** | 指向单一知识点（如"INFJ 的认知功能栈"） | 直接定位目标页面，不跟随双链延伸 |
| **关联提取** | 涉及比较、分析（如"INFJ 和 ISTP 处理冲突的区别"） | 定位主要页面后，可沿双链延伸，遵守深度限制 |
| **广域提取** | 范围宽泛（如"全面分析 INFJ"） | 从精炼总览入手，选择性延伸到子主题，避免全量阅读 |

### 双链延伸规则

页面链接已按严格层级组织（只含父级 + 同类别兄弟），不存在跨类别链接，因此延伸路径天然受限。

| 场景 | 操作 |
|------|------|
| 精确提取 | 直接定位目标页面，不跟随任何双链 |
| 关联提取 | 沿同系列兄弟链接延伸，最多 2 个 |
| 广域提取 | 从枢纽页出发，选择性进入详情页，每个系列最多读 1 个总论页 |
| 跨类别需求 | 回到 wiki-index.md 重新定位，不通过页面链接跨越类别 |

## 三、信息提取规则

### 可提取范围

`active`、`draft`、`unreviewed` 三种状态的页面均可正常提取，直接引用内容，无需附加状态注释。

`review` 状态的页面可提取，但需注明内容近期有更新。

### 内容呈现

- 客观呈现页面中的信息，保留原有标记（共识、分歧等），不做额外加工
- 知识单元中 `_` 开头的溯源字段不得纳入提取结果
- 提取综合多个页面时，末尾简要列出参考页面
- 输出中涉及 MBTI 专业术语时，须使用 `docs/MBTI术语表.md` 中的标准译法

### 知识库无法覆盖时

如实说明当前知识库未覆盖该内容，不编造信息。

## 四、查阅中的记录规则

> 以下记录操作不得影响信息提取效率。

### 需要记录的情况

| 情况 | 操作 |
|------|------|
| Wiki 和知识单元均无匹配，从知识单元合成了内容 | 将要点整理为新页面（status: `unreviewed`），在 wiki-index.md 注册，log.md 记录 |
| 提取中产出了有价值的跨页面综合分析 | 记录为 `wiki/comparisons/` 下的新页面（status: `unreviewed`），内容可简要 |
| 涉及 Wiki 中尚无独立页面的主题 | 创建占位页面（status: `unreviewed`），包含本次提取涉及的要点 |

### 不需要操作的情况

| 情况 | 处理 |
|------|------|
| 发现已有页面存在信息缺口 | 简单记录提示（不可列入被提取信息范围） |
| 提取内容与已有页面部分重叠 | 正常提取，不做任何记录 |

### 统一规则

- 所有新建页面 status 一律为 `unreviewed`
- 在 `wiki/log.md` 追加记录（格式：`## [日期] query-backfill | 简述`）
- **不修改任何已有页面**——已有页面的更新由编译流程或维护检查负责

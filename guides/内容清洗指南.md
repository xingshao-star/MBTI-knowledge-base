# 内容清洗指南

> 本指南用于清理原始资料（raw/）和知识单元（units/）中与 MBTI 专业内容无关的品牌、版权、推广等内容。
> 适用于所有来源类型。每次收录新资料时，在拆解知识单元之前先执行本指南。

---

## 一、清洗时机

| 时机 | 说明 |
|------|------|
| 新资料收录前 | 对 `raw/` 中的原始资料执行清洗，再进入拆解流程 |
| 知识单元生成后 | 校验生成的知识单元是否残留待清洗内容 |
| 存量资料补查 | 对已有 `raw/` 和 `units/` 文件进行批量扫描 |

---

## 二、待清洗内容分类

### A 类：整段移除

以下内容若构成独立段落或独立章节，直接整段删除。

| 类别 | 识别特征 | 示例 |
|------|----------|------|
| **版权声明** | 含"©"、"版权归"、"保留所有权利"、"NERIS Analytics Limited" | "所有内容版权归 ©2011-2018 NERIS Analytics Limited 所有。" |
| **法律免责** | 含"出版方不承担"、"免责"、"暗示保证"、"不构成心理咨询" | "出版方已尽最大努力编制本文件……" |
| **购买/反盗版声明** | 含"16personalities.com"、"未经授权的副本"、"购买" | "本文件仅可在 https://www.16personalities.com 购买。" |
| **网站推广** | 含"浏览我们的网站"、"文章专区"、"学院板块"、"在线学院" | "此外，请花些时间浏览我们的网站——例如文章专区……" |
| **研究推广** | 整段以推广研究资源、邀请联系为主旨 | "在过去几年中，我们开展了超过800项研究……欢迎通过我们的网站联系我们" |
| **测试引导** | 整段围绕网站测试体验展开，无 MBTI 理论价值 | "当人们在我们的网站上收到性格测试结果时……" |
| **平台方法论自述** | 整段描述 16Personalities 平台自身的模型设计决策 | "我们的模型融合了两者的优势。我们采用迈尔斯-布里格斯提出的首字母缩写格式……" |

### B 类：行内替换

以下内容穿插在有价值的段落中，仅替换相关词句，保留段落其余内容。

| 原文模式 | 替换为 | 说明 |
|----------|--------|------|
| `16Personalities 的方法论` | `MBTI 的方法论` | 品牌名→通用理论名 |
| `16Personalities 模型` / `16Personalities 的模型` | `MBTI 人格模型` | 同上 |
| `在一项 16Personalities 的研究中` | `在一项研究中` | 去除品牌名 |
| `我们的研究表明` / `我们的研究发现` / `我们的研究揭示` | `研究表明` / `研究发现` / `研究揭示` | 去除第一人称品牌视角 |
| `在我们的研究中` | `在研究中` | 同上 |
| `我们在首个量表中……我们将该量表命名为` | 改为被动句式，如"首个量表被命名为" | 去除品牌主语 |
| `我们将在本简介的后续部分` | `本简介的后续部分将` | 去除品牌主语 |
| `鉴于我们在本模型中……本文将不再深入探讨` | 删除整句 | 品牌自引用，无信息量 |

### C 类：保留不动

以下内容虽含相似关键词，但属于 MBTI 专业内容，不应清洗。

| 内容 | 保留原因 |
|------|----------|
| "个人品牌"（职业/求职语境） | 指个人职业形象，非平台品牌 |
| "为组织打造品牌"（职场协作语境） | 指组织品牌建设，非平台品牌 |
| "我们的性格"、"我们的行为"（泛指人类） | 泛指人类群体的"我们"，非品牌自称 |
| 地域分布数据中的统计数字 | 有参考价值的研究数据，保留 |

## 三、执行流程

### 第一步：关键词扫描

对目标文件执行以下关键词搜索，定位所有待处理位置：

```
扫描关键词（正则）：
16Personalities|16personalities|NERIS|版权归|保留所有权利|
我们的网站|我们的模型|我们的方法|我们的研究|我们的在线|
联系我们|在线学院|出版方|未经授权|免责
```

### 第二步：逐条分类判断

对每个匹配结果，判断属于 A 类（整段移除）、B 类（行内替换）还是 C 类（保留）：

```
匹配结果 →
├─ 所在段落整体为推广/法律/版权内容？ → A 类：整段移除
├─ 关键词穿插在有价值的 MBTI 内容中？ → B 类：行内替换
└─ 关键词在专业语境中有独立含义？     → C 类：保留不动
```

### 第三步：执行清洗

1. 先处理 A 类（整段移除），从文件末尾向前处理，避免行号偏移
2. 再处理 B 类（行内替换）
3. 跳过 C 类

### 第四步：元数据校对（仅 units/）

当知识单元文件发生内容删减时，须校对以下 frontmatter 字段：

| 字段 | 校对方式 |
|------|----------|
| `_source_lines` | 对照 `raw/` 源文件，确认剩余内容实际覆盖的行号范围 |
| `_word_count` | 重新估算清洗后的字数 |

### 第五步：验证

清洗完成后，对目标文件再次执行第一步的关键词扫描，确认无遗漏。

---

## 四、注意事项

1. **先清洗 raw，再拆解 units**：新资料收录时，优先对 `raw/` 源文件执行清洗，这样拆解出的知识单元天然干净，减少二次清理
2. **不改变段落语义**：B 类替换后，段落的核心信息和逻辑必须保持不变
3. **记录重大删减**：如果整段移除导致某个主题的论述出现断裂，在对应知识单元的 `related_units` 中标注缺口，或在收录报告中说明
4. **品牌关键词可能演变**：不同来源可能使用不同的品牌名称或网址，遇到新的品牌标识时，将其补充到本指南第二步的扫描关键词列表中
